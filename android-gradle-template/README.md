# Gradle 8.x Upgrade Templates for PerkWorth Android

> **Why this folder exists**: The Capacitor 6 project's `android/` folder is git-ignored (auto-generated by `npx cap add android`) so we can't ship the config in-repo. These templates are the exact contents you should drop into your local `android/` after generating it, to bring the project up to Gradle 8.7 + Android Gradle Plugin (AGP) 8.5.x + JDK 17 — the recommended combo for Capacitor 6 as of Feb 2026.

## Prerequisites
- **JDK 17** installed and set as `JAVA_HOME` (Android Studio Iguana bundles Zulu 17 — use that).
- **Android Studio Iguana (2023.2.1)** or newer.
- **Android SDK 34** installed via SDK Manager.

## How to apply (one-time, in your local checkout)

```bash
cd frontend
npx cap sync android              # ensures android/ exists and is up-to-date
```

Then copy each file below to the matching path inside your local `android/` folder, replacing the existing content. Commit as a single "Gradle 8 upgrade" commit AFTER you finish resolving your current merge — that way this stays out of the merge diff.

| Template file | Copy to (inside your project) |
|---|---|
| `gradle-wrapper.properties` | `android/gradle/wrapper/gradle-wrapper.properties` |
| `variables.gradle`          | `android/variables.gradle` |
| `build.gradle`              | `android/build.gradle` |
| `gradle.properties`         | `android/gradle.properties` |

Then from `android/`:
```bash
./gradlew wrapper --gradle-version=8.7 --distribution-type=all
./gradlew clean
./gradlew :app:assembleDebug
```

If the build fails on a plugin version mismatch, run `npx cap sync android` again — Capacitor 6 will normalise any drift.

## What changed vs Gradle 2.13 (your current version)

- **Gradle 2.13 → 8.7** — 8+ years of tooling improvements; 2.13 predates Android Gradle Plugin 3.0 entirely and can't build any modern Android app.
- **AGP 8.5.x** — required for `compileSdk 35` and Java 17 toolchain.
- **JDK 8 → 17** — Capacitor 6 minimum, and required by AGP 8.
- **Gradle Kotlin DSL not switched to** — kept Groovy DSL because the Capacitor CLI still generates `.gradle` files (not `.gradle.kts`). Migrating is possible but out of scope for a "just get the build working" upgrade.

## Rollback

If anything breaks, `npx cap add android --force` will regenerate the entire `android/` folder from Capacitor's default templates. You'll lose signing config + custom edits, so back them up first.
